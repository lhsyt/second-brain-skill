# 内容创作者第二大脑 / Second Brain for Content Creators

**把你每天写的东西，自动沉淀成可复用的内容资产。**

一个跑在 Claude Code 上的 skill。你只管写，AI 自动整理、提炼、归档。

---

## 5 分钟快速上手

### 前提条件

- 已安装 [Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview)（Anthropic 官方 CLI 工具）
- 有一个你平时写东西的文件夹（或者新建一个也行）

### 第一步：安装 skill

打开终端，复制粘贴这三行：

```bash
mkdir -p ~/.claude/skills/second-brain
curl -o ~/.claude/skills/second-brain/SKILL.md \
  https://raw.githubusercontent.com/canoe-tool/second-brain-skill/main/SKILL.md
```

> 这会把 skill 文件下载到 Claude Code 的全局 skill 目录。安装一次，所有项目都能用。

### 第二步：建立你的项目文件夹

```bash
mkdir -p my-second-brain/{00-inbox/insight-cards,03-resources/{atoms,sources}}
cd my-second-brain
```

然后下载个人档案模板：

```bash
curl -o profile.md \
  https://raw.githubusercontent.com/canoe-tool/second-brain-skill/main/profile-template.md
```

### 第三步：填写你的档案

打开 `profile.md`，填写你的信息。**重点填这几个：**

| 字段 | 为什么重要 |
|------|-----------|
| 写作风格 | 决定 AI 整理文章时保留什么、不改什么 |
| 标志性表达 | AI 看到这些词会原封不动保留 |
| 绝对不写 | AI 绝不会在输出中使用这些表达 |

> **不知道怎么描述自己的风格？** 把你写过的 5-10 篇内容贴给 Claude，说"帮我分析我的写作风格，填到 profile.md 里"。AI 总结你的风格比你自己描述更准。

### 第四步：喂入你的历史内容（可选但推荐）

如果你有已经写过的文章/日记/笔记，把它们放进项目文件夹，然后告诉 Claude：

```
帮我把这些旧内容整理进资产库
```

Claude 会扫描所有内容，自动提取故事、洞见、金句，建立你的初始资产库。

**这一步的价值：** 你可能已经写了几十万字的内容，里面埋着大量可复用的好东西——但它们分散在各处，你自己都忘了。AI 帮你挖出来、分类好、编号好，以后随时调用。

### 第五步：开始日常使用

在项目目录下打开 Claude Code：

```bash
cd my-second-brain
claude
```

然后直接把你今天写的东西贴进去：

```
这是我今天的日记：

[你的内容]
```

Claude 会自动走完整流程：整理文章 → 发你审核 → 提取资产 → 生成洞见卡片 → 给教练反馈。

## 它具体做什么

每次你发来一篇内容，Claude 执行 6 步流程：

### Step 1：理解输入
区分口述（修正语音识别错误）和手写（只调明显口语化表达）。口述内容有重复、口误、逻辑跳跃是正常的，AI 不会当作错误处理。

### Step 2：给标题
3-5 个标题选项。短、有力、有观点。

### Step 3：整理文章
**核心原则：最小改动，保留你的声音。**
- 只改口头语重复和语音识别错误
- 句式结构不动——你的节奏就是你的风格
- 不加小标题、不删内容、不重写
- **整理完必须先发你审核，你说"可以"才继续**

> 为什么要审核？因为 AI 可能误判你的意图。你是唯一有权判断"这还是我"的人。

### Step 4：内容资产提取
审核通过后，AI 从这篇内容里自动提取 4 类可复用资产：

| 资产类型 | 是什么 | 怎么复用 |
|---------|--------|---------|
| **故事** | 有场景、冲突、转折的具体经历 | 下次写文章时当开头或论据 |
| **洞见** | 反直觉、有解释力、来自真实经历的判断 | 做选题、写观点文、拍短视频的核心论点 |
| **风格表达** | "一看就知道是你写的"标志性原句 | 保存你的语言指纹，帮 AI 学你的腔调 |
| **动作** | 读者 10 分钟内能做的具体步骤 | 转化率最高的内容类型，单独存方便调用 |

所有资产自动编号、自动存档、跨对话不断档。

### Step 5：生成洞见卡片
把当天最核心的洞见做成一张可下载的 HTML 卡片。

### Step 6：教练反馈
- 今日观察：这篇是突破还是在重复老话题？
- 做得好的地方（具体的，不是"你很棒"）
- 值得留意的地方
- 一个挑战问题

## 快捷指令

| 你说 | Claude 做什么 |
|------|-------------|
| 「这是我的日记」/「今天的文章」 | 完整 6 步流程 |
| 「整理一下」 | 只做标题+文章，等审核 |
| 「下一步」（审核通过后） | 执行资产提取+卡片+反馈 |
| 「不用改，按原文」 | 原文存档，直接提取资产 |
| 「改太多了」 | 退回最小改动版本，重新审核 |
| 「分析一下」 | 只分析，不走流程 |

## 文件结构

```
my-second-brain/
├── profile.md                  ← 你的个人档案（填一次，持续更新）
├── 00-inbox/
│   └── insight-cards/          ← 每天的洞见卡片（HTML）
├── 03-resources/
│   ├── atoms/
│   │   ├── story-bank.md       ← 故事库（自动编号 #001, #002...）
│   │   ├── insight-bank.md     ← 洞见库（IN-001, IN-002...）
│   │   ├── action-bank.md      ← 动作库（ACT-001, ACT-002...）
│   │   └── style-bank.md       ← 风格表达库（STY-001, STY-002...）
│   └── sources/                ← 每篇内容的定稿存档
└── [其他你自己的文件]
```

> 这些文件不需要你手动创建。第一次使用时 Claude 会自动建好。

## 背后的逻辑

### 为什么要拆成 4 个库？

一篇文章里有不同类型的价值。混在一起存没用，因为你不知道该找什么。

拆开来的好处：
- 要写一篇关于"坚持"的文章？→ 去故事库搜所有关于"坚持"的故事，挑 2-3 个串起来
- 要做短视频？→ 去洞见库找一个反直觉观点当钩子
- 要发朋友圈？→ 去风格库里挑一句金句

**资产库就是你的内容原材料仓库。** 写得越久，库存越多，创作越快。

### 为什么不让 AI 直接写文章？

因为 AI 写出来的东西有"AI 味"——通顺但没有个性。

这个系统的思路相反：**你来写，AI 来整理。** 你的声音、你的故事、你的表达方式——这些是 AI 写不出来的。AI 做的是你不想做但必须做的苦力活：分类、编号、归档、提取。

而且资产库本身也在训练 AI——它读了你几百条故事和几十条风格表达之后，输出的内容会越来越像你。

### 为什么用 Claude Code？

两个原因：
1. **自动读文件**：Claude Code 每次打开会自动读取项目里的文件，不用手动粘贴提示词
2. **自动写文件**：提取完资产，直接写进 story-bank.md，编号自动接续。你不用手动复制粘贴

> 但你的资产库是 Markdown 文件，任何工具都能读。如果哪天换了工具，资产库跟着你走。

## 进阶用法

### 母题系统

写了 50 篇以上后，你会发现自己反复在聊几个核心主题。把它们提炼出来就是"母题"。

告诉 Claude：`帮我分析资产库，找出反复出现的主题，建立母题库`

母题让你的创作有体系感——不是每天想"写什么"，而是在几个核心主题之间轮转。

### Concept Handle

给每条洞见起一个 4 字以内的名字，让它好传播好记忆。

比如"过路费思维""能力盲区""AI天花板定律"——这些名字让抽象观点变成可以口口相传的概念。

### 资产复用

故事库积累到 100+ 条后：

```
用故事库里关于"坚持"的故事，帮我写一篇公众号文章
```

AI 从库里调取相关故事，串成新文章——用的全是你自己的真实经历和表达方式。

## 常见问题

**Q：我不日更，能用吗？**
能。一天一篇还是一周一篇，流程一样。日更的人资产库长得更快而已。

**Q：口述和手写都行吗？**
都行。口述的话先用手机语音转文字，把文字贴给 Claude。AI 会自动修正语音转文字的错误。

**Q：能跟 Obsidian 一起用吗？**
能。把项目文件夹设成 Obsidian 的 vault，Markdown 文件在 Obsidian 里也能浏览和搜索。

**Q：我的内容会上传到哪里？**
文件都在你本地电脑上。Claude Code 是本地运行的，你的内容不会离开你的电脑。
